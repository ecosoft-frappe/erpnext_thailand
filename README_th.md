# ERPNext Thailand

ส่วนเสริม ERPNext สำหรับประเทศไทย — รองรับภาษีมูลค่าเพิ่ม ภาษีหัก ณ ที่จ่าย รายงานสรรพากร และกระบวนการทางธุรกิจตามกฎหมายไทย

---

## คุณสมบัติ

### 1. ภาษีมูลค่าเพิ่ม (VAT)

- **ภาษีสินค้า**: จุดรับรู้ภาษีเกิดขึ้นเมื่อส่งมอบสินค้าและออกใบแจ้งหนี้ ระบบสร้างใบกำกับภาษีซื้อ/ขายโดยอัตโนมัติเมื่อมีรายการบัญชี (GL Entry)
- **ภาษีบริการ (ภาษีค้างชำระ)**: จุดรับรู้ภาษีเกิดขึ้นเมื่อชำระเงินหรือออกใบเสร็จรับเงิน บันทึก GL เป็น "ภาษีค้างชำระ" ตอนออกใบแจ้งหนี้ และโอนเป็นภาษีจริงเมื่อมีการชำระเงิน
- **ภาษีในใบเบิกค่าใช้จ่าย (Expense Claim)**: สามารถบันทึกรายละเอียดใบกำกับภาษีในใบเบิกค่าใช้จ่ายได้โดยตรง
- **ภาษีในสมุดรายวัน (Journal Entry)**: รองรับการบันทึกรายละเอียดใบกำกับภาษีในสมุดรายวัน
- **รองรับภาษี 0%**: สำหรับการส่งออกและธุรกรรมที่ได้รับการยกเว้น

### 2. ภาษีหัก ณ ที่จ่าย และใบรับรองการหักภาษี ณ ที่จ่าย

- **หักภาษี ณ ที่จ่ายในการชำระเงิน**: หักภาษีในขั้นตอนการบันทึกรายการชำระเงิน (Payment Entry) เช่น 1%, 2%, 3%, 5%
- **ตรวจจับประเภท WHT อัตโนมัติจากสินค้า**: ระบบอ่านประเภทการหักภาษีจากรายการสินค้าในใบแจ้งหนี้ โดยแยกอัตราสำหรับบุคคลธรรมดาและนิติบุคคลโดยอัตโนมัติ
- **ประเภทเงินได้**: กำหนดรหัสประเภทเงินได้สำหรับแบบ PND (ประเภทที่ 1–6 รวมถึงประเภทย่อย 4.1.x และ 4.2.x)
- **ใบรับรองการหักภาษี ณ ที่จ่าย**: สร้างและพิมพ์ใบรับรองฯ จากรายการชำระเงิน พร้อมชื่อผู้จัดจำหน่าย ที่อยู่ ประเภทเงินได้ และจำนวนภาษี

### 3. รายงานสำหรับกรมสรรพากร

- **รายงานภาษีขาย**: สร้างรายงาน PDF หรือ Excel สำหรับยื่นกรมสรรพากร
- **รายงานภาษีซื้อ**: สร้างรายงาน PDF หรือ Excel สำหรับยื่นกรมสรรพากร
- **รายงาน PND3**: รายงานภาษีหัก ณ ที่จ่าย สำหรับผู้รับที่เป็นบุคคลธรรมดา
- **รายงาน PND53**: รายงานภาษีหัก ณ ที่จ่าย สำหรับผู้รับที่เป็นนิติบุคคล

### 4. ใบวางบิล (Sales/Purchase Billing)

- **ใบวางบิลขาย**: รวบรวมใบแจ้งหนี้ขายหลายใบเป็นเอกสารวางบิลฉบับเดียวสำหรับลูกค้า
- **ใบวางบิลซื้อ**: รวบรวมใบแจ้งหนี้ซื้อหลายใบจากผู้จัดจำหน่าย
- **ใบรับเงิน (Payment Receipt)**: ตั้งค่าให้สร้างใบรับเงินโดยอัตโนมัติเมื่อบันทึกการรับชำระเงินจากใบวางบิลขาย
- **ติดตามยอดคงค้าง**: เอกสารใบวางบิลแสดงยอดรวมค้างชำระและปิดอัตโนมัติเมื่อชำระครบ

### 5. การจัดการเงินมัดจำ (Deposit Invoice)

- **สร้างใบแจ้งหนี้มัดจำ**: สร้างใบแจ้งหนี้แรกเป็นใบแจ้งหนี้มัดจำโดยตรงจากใบสั่งขายหรือใบสั่งซื้อ
- **การจัดสรรมัดจำอัตโนมัติ**: ใบแจ้งหนี้ถัดไปสำหรับคำสั่งเดิมจะหักยอดมัดจำโดยอัตโนมัติ โดยใช้วิธีตามเปอร์เซ็นต์ (Percent) หรือหักเต็มจำนวน (Full Amount)
- **มัดจำที่ไม่ผูกกับคำสั่ง**: มัดจำที่ไม่ได้ผูกกับคำสั่งซื้อ/ขายสามารถนำมาหักกลบกับใบแจ้งหนี้ของลูกค้า/ผู้จัดจำหน่ายรายเดิมได้
- **ติดตามยอดมัดจำคงเหลือ**: ระบบติดตามยอดมัดจำคงเหลือในทุกใบแจ้งหนี้ที่ตามมา
- **การจัดสรรด้วยตนเอง**: ผู้ใช้สามารถเปลี่ยนเป็นโหมดจัดสรรมัดจำด้วยตนเองเมื่อต้องการ

### 6. การจัดการเงินสดย่อย (Petty Cash)

- **ผู้รับผิดชอบเงินสดย่อย (Petty Cash Holder)**: กำหนดผู้ดูแลเงินสดย่อย ผูกกับบัญชีเงินสดย่อยและวงเงินที่ได้รับ
- **บัญชีเงินสดย่อย**: ทำเครื่องหมายบัญชีใดก็ได้ว่าเป็นบัญชีเงินสดย่อย ระบบจะตรวจสอบตอนบันทึกการชำระเงิน
- **เติมเงินและถอนเงิน**: สร้างสมุดรายวันสำหรับการเติมเงินหรือถอนเงินจากผู้ดูแลได้โดยตรงจากฟอร์ม
- **รายงานเงินสดย่อย**: รายงานสรุปรายการเงินสดย่อยแยกตามผู้ดูแล
- **เชื่อมต่อกับใบเบิกค่าใช้จ่าย**: ใบเบิกค่าใช้จ่ายสามารถทำเครื่องหมายเป็นเงินสดย่อย โดยผูกผู้ดูแลและบัญชีเงินสดย่อยตลอดกระบวนการชำระเงิน

### 7. การตั้งค่าภาษีไทย

- **ตั้งค่าแบบรวมศูนย์**: กำหนดบัญชีภาษีไทยทั้งหมดในที่เดียว (Thai Tax Settings)
- **รองรับหลายบริษัท**: แต่ละบริษัทสามารถมีการตั้งค่าบัญชีภาษีแยกกันได้
- **ที่อยู่ภาษี**: จัดการที่อยู่สำหรับออกใบกำกับภาษีของบริษัท

### 8. การรองรับภาษาไทย

- **จำนวนเงินเป็นตัวอักษรไทย**: แปลงตัวเลขเป็นข้อความภาษาไทย (เช่น "หนึ่งร้อยบาทถ้วน") สำหรับใช้ในฟอร์มพิมพ์
- **จำนวนเงินหลายสกุล**: แปลงจำนวนเงินเป็นตัวอักษรไทยหรืออังกฤษตามสกุลเงินของธุรกรรม
- **วันที่ตามปฏิทินไทย**: แสดงวันที่ในรูปแบบไทยเต็ม (เช่น `30 ตุลาคม 2560`) สำหรับฟอร์มพิมพ์

### 9. อัตราแลกเปลี่ยนจากธนาคารแห่งประเทศไทย (BOT)

- **เชื่อมต่อ BOT API**: ดึงอัตราแลกเปลี่ยนประจำวัน (ค่าเฉลี่ย) จาก API ของธนาคารแห่งประเทศไทย (gateway.api.bot.or.th)
- **Token แบบปลอดภัย**: เก็บ API Token ไว้ใน Currency Exchange Settings อย่างปลอดภัย
- **ค้นหาย้อนหลัง**: ระบบค้นหาย้อนหลังสูงสุด 5 วันเพื่อหาอัตราล่าสุดเมื่อวันที่ระบุไม่มีข้อมูล

### 10. การพิมพ์เอกสาร

- **จำนวนสำเนา**: กำหนดจำนวนสำเนาที่ต้องพิมพ์ (ต้นฉบับ + สำเนาเพิ่มเติม) โดยลำดับสำเนาสามารถใช้ใน Jinja เพื่อพิมพ์ข้อความ เช่น "ต้นฉบับ" หรือ "สำเนา"
- **เลือก Print Format อัตโนมัติ**: กำหนดเงื่อนไข (Jinja expression) บนแต่ละ Print Format ระบบจะเลือก Format ที่เหมาะสมกับเอกสารโดยอัตโนมัติ
- **ซ่อน Format ที่ไม่ตรงเงื่อนไข**: Print Format ที่ไม่ตรงกับบริบทของเอกสารปัจจุบันสามารถซ่อนออกจากรายการได้

### 11. บริการดึงที่อยู่ไทย

- **ค้นหาตามรหัสไปรษณีย์**: สืบค้นฐานข้อมูลรหัสไปรษณีย์ไทยเพื่อกรอกตำบล อำเภอ และจังหวัดในฟอร์มที่อยู่โดยอัตโนมัติ
- **ค้นหาตามเลขประจำตัวผู้เสียภาษี**: เรียกใช้บริการเว็บของกรมสรรพากร (rdws.rd.go.th) ด้วยเลขประจำตัวผู้เสียภาษีและรหัสสาขาเพื่อดึงที่อยู่ที่จดทะเบียน
- **ซิงค์ที่อยู่ไปยังลูกค้า/ผู้จัดจำหน่าย**: เมื่ออัปเดตที่อยู่ด้วยเลขประจำตัวผู้เสียภาษีหรือรหัสสาขาใหม่ ระบบจะอัปเดตข้อมูลใน Customer หรือ Supplier ที่เชื่อมโยงโดยอัตโนมัติ

### 12. การตั้งชื่อเอกสารตามวันที่

- **ตัวแปรชื่อเอกสารแบบวันที่**: ตัวแปรเพิ่มเติมสำหรับ Naming Series โดยใช้ `posting_date` หรือ `transaction_date` ของเอกสาร:
  - `YYYY-DATE` — ปี 4 หลัก
  - `YY-DATE` — ปี 2 หลัก
  - `MM-DATE` — เดือน 2 หลัก
  - `DD-DATE` — วัน 2 หลัก
  - `WW-DATE` — หมายเลขสัปดาห์ต่อเนื่อง

## การตั้งค่า

### ตั้งค่าใบกำกับภาษี (VAT)

1. ในผังบัญชี ให้สร้างบัญชีต่อไปนี้พร้อมอัตราภาษี (เช่น 7%):
   - ภาษีขาย (Sales Tax) และภาษีขายค้างชำระ (Undue Sales Tax)
   - ภาษีซื้อ (Purchase Tax) และภาษีซื้อค้างชำระ (Undue Purchase Tax)
2. เปิด **Thai Tax Settings** และผูกแต่ละบริษัทกับบัญชีภาษีข้างต้น
3. ตั้งค่า **Sales/Purchase Taxes and Charges Templates** เพื่อให้:
   - การซื้อ/ขายสินค้า บันทึก GL ไปยังบัญชีภาษีทันที
   - การซื้อ/ขายบริการ บันทึก GL ไปยังบัญชีภาษีค้างชำระ แล้วโอนเป็นภาษีจริงเมื่อชำระเงิน
4. ตรวจสอบว่าบริษัทมี **ที่อยู่วางบิล (Company Billing Address)** ที่จะปรากฏบนใบกำกับภาษี
5. ตรวจสอบว่า **ลูกค้า** และ **ผู้จัดจำหน่าย** ทุกรายมีที่อยู่วางบิลพร้อมเลขประจำตัวผู้เสียภาษี

เมื่อตั้งค่าครบแล้ว ระบบจะสร้างใบกำกับภาษีซื้อ/ขายโดยอัตโนมัติทุกครั้งที่มีการบันทึกรายการภาษี

### ตั้งค่าภาษีหัก ณ ที่จ่าย

1. ในผังบัญชี สร้างบัญชี **ภาษีหัก ณ ที่จ่าย** (Withholding Tax Payable)
2. สร้าง **Withholding Tax Types** (เช่น 1%, 2%, 3%, 5%) และผูกกับบัญชี WHT ของแต่ละบริษัท
3. กำหนดประเภท WHT ให้กับ **Item** (แยกสำหรับบุคคลธรรมดาและนิติบุคคล)

ในขั้นตอนการบันทึกรายการชำระเงิน ระบบจะตรวจจับประเภท WHT จากรายการสินค้าในใบแจ้งหนี้โดยอัตโนมัติ ผู้ใช้สามารถคลิก **Create Withholding Tax Cert** เพื่อออกใบรับรองการหักภาษีได้ทันที

### ตั้งค่าเงินสดย่อย

1. ทำเครื่องหมาย **Is Petty Cash Account** ในฟอร์มบัญชีที่ต้องการ
2. สร้าง **Petty Cash Holder** ผูกกับบัญชีเงินสดย่อยและกำหนดวงเงิน
3. ใช้ปุ่ม **Top Up** และ **Withdraw** ในฟอร์ม Petty Cash Holder เพื่อโอนเงินเข้า-ออก
4. ในใบเบิกค่าใช้จ่ายและรายการชำระเงิน เลือก **Is Petty Cash** และระบุผู้ดูแลเพื่อผ่านรายการไปยังบัญชีเงินสดย่อย

### ตั้งค่าใบแจ้งหนี้มัดจำ

1. สร้าง **Deposit Item** (ตั้งค่า `is_deposit_item = 1`) พร้อมบัญชีรายได้/รายจ่ายที่เหมาะสม
2. ในใบสั่งขายหรือใบสั่งซื้อ เปิดใช้ **Has Deposit** และเลือกวิธีหัก (Percent หรือ Full Amount)
3. จากคำสั่งซื้อ/ขาย คลิก **Create Deposit Invoice** เพื่อออกใบแจ้งหนี้มัดจำแรก
4. ใบแจ้งหนี้ถัดไปของคำสั่งเดิมจะแสดงรายการหักมัดจำโดยอัตโนมัติ

### ตั้งค่าอัตราแลกเปลี่ยนจาก BOT

1. ลงทะเบียนขอ API Token จากพอร์ทัลนักพัฒนาของธนาคารแห่งประเทศไทย
2. ใน **Currency Exchange Settings** เลือก **BOT** เป็นแหล่งข้อมูลและกรอก API Token
